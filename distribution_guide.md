# 임하밍 모음 (imhaming-moeum) 배포 및 공유 가이드

이 문서는 현재 개발된 웹(Web)과 데스크톱 애플리케이션(App)을 다른 사람이 사용할 수 있도록 배포(공유)하는 방법을 단계별로 안내합니다.

---

## 🖥️ 1. 데스크톱 앱 (PC 버전) 배포 방법

데스크톱 앱은 Electron을 기반으로 작동하며, `electron-builder`를 사용해 압축 파일(.zip)이나 설치 파일로 만들어 다른 사람에게 전달할 수 있습니다.

### 빌드(Build) 방법

1. 터미널(명령 프롬프트)을 열고 현재 프로젝트 폴더(`imhaming`)로 이동합니다.
2. 아래 명령어를 실행하여 데스크톱 앱을 빌드합니다:
   ```bash
   npm run electron:build
   ```
3. 빌드가 완료되면 프로젝트 폴더 안에 `release-build-v2` 폴더가 생성됩니다.
4. 해당 폴더 안에 있는 `release-build-v2.zip` (또는 폴더 내의 실행 파일)을 다른 사람에게 메일이나 드라이브, 메신저 등으로 전달합니다.

### 다른 사람이 사용하는 방법

- 전달받은 `.zip` 파일의 압축을 풉니다.
- 압축 해제된 폴더 안에 있는 `imhaming-moeum.exe` (또는 관련 실행 파일)를 더블 클릭하여 실행합니다.
- (참고: 별도의 웹 서버나 코드 설치 없이 바로 실행됩니다.)

---

## 🌐 2. 웹 브라우저 (Web) 버전 배포 방법

웹 버전은 링크 주소(URL)만 상대방에게 알려주면 누구나 인터넷 브라우저로 접속해 사용할 수 있습니다. 현재 설정에 맞춰 두 가지 방법이 가능합니다.

### 방법 A: Vercel을 이용한 배포 (추천)

`vercel.json`이 설정되어 있으므로 Vercel을 통해 매우 쉽게 사이트를 배포할 수 있습니다.

1. [Vercel](https://vercel.com/) 사이트에 회원가입/로그인합니다.
2. 현재 프로젝트를 GitHub에 업로드(Push)합니다.
3. Vercel에서 'Add New Project'를 선택하고, 방금 업로드한 GitHub 저장소(Repository)를 연결합니다.
4. Framework Preset을 `Vite`로 설정하고 [Deploy] 버튼을 누릅니다.
5. 배포가 완료되면 발급되는 `.vercel.app` 링크를 사람들에게 공유합니다.

### 방법 B: Base44 (백엔드 통합 배포)

현재 `package.json`에 Base44 스크립트가 포함되어 있습니다. CLI를 통해 자체 배포가 가능합니다.

1. 터미널에서 Base44에 로그인합니다:
   ```bash
   npm run base44:login
   ```
2. 웹사이트를 빌드하고 배포합니다:
   ```bash
   npm run base44:deploy
   ```
3. 배포가 끝나면 터미널에 표시되는 호스팅 주소(URL)를 복사하여 공유합니다.

## 🛠️ 3. 다른 사람 버전을 만들 때 참고/주의사항

### 1) 데이터베이스 (DB) 분리 혹은 공유

- 현재 Base44(또는 Supabase) 백엔드를 연동하여 사용하고 있는 경우, **모든 사용자가 같은 데이터베이스를 바라보게 됩니다**.
- 만약 "다른 사람도 각자의 데이터만 관리하는 완전히 독립적인 프로그램"으로 만들고 싶다면:
  - 사용자별 로그인을 구현하여(Auth) 데이터 접근 권한을 나누거나,
  - 해당 사용자를 위한 별도의 DB(Base44 프로젝트)를 새로 만들어 `.env` 혹은 클라이언트 키를 교체하여 새로 빌드해야 합니다.

### 2) 중복 실행 방지 기능

- 최근에 적용된 '앱 중복 실행 방지(Single Instance Lock)' 코드가 데스크톱 앱 버전에 포함되어 있습니다.
- 따라서 다른 사람이 실행 파일을 여러 번 더블클릭하더라도, 프로그램은 한 번만 열리며 기존 창이 화면 앞으로 튀어나오게 됩니다. 해당 기능은 빌드 시 자동 적용됩니다.

### 3) 윈도우(Windows) 보안 경고 (SmartScreen)

- 데스크톱 앱 빌드 후 전달 시, 공식 인증서(Code Signing)로 서명하지 않았기 때문에 다른 사람의 PC에서 실행 시 **"Windows의 PC 보호 (안전하지 않은 프로그램)"** 경고창이 뜰 수 있습니다.
- 이때는 당황하지 말고, **[추가 정보]** 글씨를 클릭한 뒤 **[실행]** 버튼을 누르면 정상적으로 앱을 이용할 수 있다고 안내해 주면 됩니다.

### 4) 환경변수(.env) 포함 여부 (보안 주의)

- 발급받은 API 키나 데이터베이스 연결 비밀번호 등이 하드코딩 되어 있거나 `.env`에 들어있을 경우 주의가 필요합니다.
- 특히 퍼블릭 GitHub 저장소에 코드를 올릴 때는 절대 보안 키가 포함된 파일을 올리지 않도록 `.gitignore` 설정이 잘 되어있는지 확인하세요.
